Grid-X Unified Guide

A distributed federated learning system architected for collaborative machine learning across isolated provider nodes without raw data exposure.

Federated Learning Utilizes FedAvg algorithm to aggregate local gradients into a global model.
Secure Execution Docker-based sandboxing ensures user scripts cannot access provider hosts.
Distributed Storage Supabase integration for robust, high-availability file and model management.
Automated Partitioning Intelligent CSV splitting distributes data shards across the available mesh.

System Architecture

The Grid-X lifecycle ensures data privacy by moving compute to the data, rather than the data to a central server.

[Image of federated learning architecture]

Core Installation

Follow these steps to initialize the Control Plane (Backend) and Data Plane (Worker).

1. Repository Initialization

git clone https://github.com/YOUR_USERNAME/Grid-X.git
cd Grid-X
python3 -m venv .venv
source .venv/bin/activate

2. Dependency Mapping

# Install Protocol Layers
pip install -r backend/requirements.txt
pip install -r worker/requirements.txt

3. Execution Environment

chmod +x build_docker.sh
./build_docker.sh

Environment Configuration

A .env file is required for backend orchestration and cloud storage connectivity.

SUPABASE_URL=your_project_url
SUPABASE_SERVICE_ROLE_KEY=your_service_key
SUPABASE_BUCKET_NAME=gridx-files

Worker Setup Guide

Provision your idle hardware to the mesh. The worker agent manages polling and Docker lifecycle.

# Initialize Worker Node
export BACKEND_URL="http://localhost:8000"
export AGENT_ID="node_alpha_01"
python worker/main.py

Configure hardware limits in worker_config.env to prevent host exhaustion.

Security Protocol

  • Data At Rest: AES-256 encryption on Supabase storage buckets.
  • Data In Flight: All model weights are transferred via TLS-encrypted streams.
  • Sandbox: Non-privileged Docker execution prevents container escape; network access is disabled during exec.
